Searching Algorithms Learn & Visualize Linear and Binary Search

Linear Search

Overview: Linear Search is a simple searching algorithm that checks each element of a list sequentially until the desired element is found or the list ends. It does not require the list to be sorted.

Use Cases: Useful for small datasets, unsorted lists, or when quick implementation is needed. Not efficient for large datasets as it has O(n) time complexity.

Algorithm Steps:

  1. Start from the first element of the array.
  2. Compare the target element with the current element.
  3. If it matches, return the index.
  4. If not, move to the next element and repeat.
  5. If the end of the array is reached without a match, return -1 (not found).

Example: Searching for 7 in [3, 5, 2, 7, 1] will check 3 → 5 → 2 → 7 and find it at index 3.

Binary Search

Overview: Binary Search is an efficient algorithm for finding an element in a sorted array. It repeatedly divides the search interval in half, reducing the search space quickly.

Use Cases: Large datasets where the array/list is sorted. It has a time complexity of O(log n), making it much faster than Linear Search for big data.

Algorithm Steps:

  1. Initialize the first (low) and last (high) indices of the array.
  2. Find the middle index: mid = Math.floor((low + high)/2).
  3. If array[mid] equals the target, return mid.
  4. If target < array[mid], search the left half (high = mid - 1).
  5. If target > array[mid], search the right half (low = mid + 1).
  6. Repeat until the target is found or low > high (not found).

Example: Searching for 7 in [1, 2, 3, 5, 7, 9] checks mid → left/right halves → finds 7 at index 4.
