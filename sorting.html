Sorting Algorithms Learn & Visualize Common Sorting Techniques

Bubble Sort

Overview: Bubble Sort repeatedly compares adjacent elements and swaps them if they are in the wrong order. Simple but inefficient for large datasets.

Use Cases: Educational purposes, small datasets, easy to implement.

Algorithm Steps:

  1. Compare each pair of adjacent elements in the array.
  2. Swap them if they are in the wrong order.
  3. Repeat for each element in the array until sorted.

Example: Sorting [5, 2, 9, 1] → pass 1: [2, 5, 1, 9], pass 2: [2, 1, 5, 9], pass 3: [1, 2, 5, 9]

Selection Sort

Overview: Selection Sort finds the minimum element from the unsorted portion and places it at the beginning. Repeats for the rest of the array.

Use Cases: Small arrays, minimal writes compared to Bubble Sort.

Algorithm Steps:

  1. Find the minimum element in the unsorted part of the array.
  2. Swap it with the first element of the unsorted part.
  3. Move the boundary of unsorted part by one and repeat.

Example: Sorting [4, 2, 5, 1] → pass 1: [1, 2, 5, 4], pass 2: [1, 2, 5, 4], pass 3: [1, 2, 4, 5]

Insertion Sort

Overview:Insertion Sort is a simple comparison-based sorting algorithm that builds the sorted array one element at a time by inserting each new element into its correct position.

Use Cases: Small datasets, nearly sorted arrays, and real-time systems. Performs efficiently when the number of elements is low. Best-case time complexity O(n), worst-case O(n²).

Algorithm Steps:

  1. Start from the second element of the array.
  2. Compare the current element with the previous elements.
  3. Shift larger elements one position to the right.
  4. Insert the current element at its correct position.
  5. Repeat until the entire array is sorted.

Example: Sorting [4, 1, 5, 2] → [1, 4, 5, 2] → [1, 4, 5, 2] → [1, 2, 4, 5]

Merge Sort

Overview: Merge Sort is a divide-and-conquer algorithm that divides the array into halves, sorts them, and merges them back.

Use Cases: Large datasets, linked lists, external sorting. Time complexity O(n log n).

Algorithm Steps:

  1. Divide the array into two halves.
  2. Recursively sort each half.
  3. Merge the sorted halves into a single sorted array.

Example: Sorting [4, 1, 5, 2] → divide: [4,1] & [5,2] → sort: [1,4] & [2,5] → merge: [1,2,4,5]

Quick Sort

Overview: Quick Sort selects a pivot and partitions the array into elements less than and greater than the pivot. Then recursively sorts the partitions.

Use Cases: Large datasets, efficient in practice. Average time complexity O(n log n), worst case O(n²).

Algorithm Steps:

  1. Pick a pivot element.
  2. Partition the array: elements smaller than pivot left, larger right.
  3. Recursively apply Quick Sort to left and right partitions.

Example: Sorting [3,6,2,5] → pivot 5 → partition: [3,2] & [6] → sort left: [2,3] → final: [2,3,5,6]

Heap Sort

Overview: Heap Sort builds a max heap from the array and repeatedly extracts the maximum element to sort the array.

Use Cases: Large datasets, in-place sorting, no recursion required for iterative heap sort. Time complexity O(n log n).

Algorithm Steps:

  1. Build a max heap from the input array.
  2. Swap the first (maximum) element with the last element.
  3. Reduce the heap size by 1 and heapify the root.
  4. Repeat until the heap size is 1.

Example: Sorting [4,10,3,5] → build max heap: [10,5,3,4] → swap 10 & 4 → heapify → repeat → [3,4,5,10]
